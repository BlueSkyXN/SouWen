## 📝 技术规范遵守

### TOCN 协议规范

#### ✅ Python 文件（文件头 + 函数 + 逻辑）

```python
"""
文件用途：xxx

函数/类清单：
    function_name(params) → return_type
        - 功能：xxx
        - 输入：xxx
        - 输出：xxx

关键逻辑注释：
    # 解释为什么这样做，而非什么代码在做
"""
```

#### ✅ Shell 脚本（顶部 + 块注释）

```shell
# ===== 用途说明 =====
# 支持的模式、环境变量清单、使用方式

# ===== 核心功能模块 =====
# 为每个大的功能块添加分隔和说明
```

#### ✅ YAML/TOML/Dockerfile（Section 注释）

```yaml
# ===== Section 名称 =====
# 本 section 的用途说明
field: value  # 字段用途补充（如需要）
```
